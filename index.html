Життєвий цикл ПЗ

Життєвий цикл програмного забезпечення (ПЗ) – це послідовність етапів розробки програмного продукту від моменту появи ідеї до його завершення та подальшого обслуговування. Основні етапи життєвого циклу ПЗ включають:

  1. Визначення вимог: Аналіз потреб користувачів, визначення функціональності, яку має забезпечити ПЗ.
  2. Проектування: Створення архітектури та структури системи. Розробляються загальні принципи і деталі майбутнього ПЗ, включаючи дизайн інтерфейсу.
  3. Розробка: Написання коду відповідно до вимог і проектної документації.
  4. Тестування: Перевірка правильності роботи ПЗ, виявлення і виправлення помилок.
  5. Впровадження: Установка та налаштування ПЗ у користувачів.
  6. Супровід: Виправлення помилок, додавання нових функцій та оптимізація роботи ПЗ.

Проектування

Проектування – це важливий етап життєвого циклу, під час якого визначається, як саме буде реалізована система. Включає в себе:

  1. Архітектурне проектування: Структура системи, її модулі та зв'язки між ними.
  2. Деталізоване проектування: Деталі реалізації окремих компонентів системи. Проектування є ключовим, оскільки помилки на цьому етапі можуть призвести до труднощів у розробці та експлуатації ПЗ.

IDEF0

IDEF0 – це методологія для моделювання функціональних процесів та систем, широко використовується в проектуванні та аналізі систем. Вона дозволяє:

  1. Моделювати і описувати структуру системи або процесу на рівні функцій.
  2. Візуально представити, як працюють окремі частини системи та які ресурси вони використовують.

Основними елементами IDEF0 є блоки (функції) та стрілки (інформаційні потоки, механізми, управління та виходи). Ця діаграма допомагає розробникам і користувачам краще зрозуміти, як система функціонує та як її компоненти взаємодіють.

IDEF0 часто використовується на ранніх етапах проектування для побудови високорівневих моделей процесів або систем.

Питання 10: CSS: Загальні поняття

CSS (Cascading Style Sheets) — це мова стилів, яка використовується для опису вигляду та формату документа, написаного на HTML. CSS дозволяє відокремити контент (структуру) від стилю, що робить розробку веб-сайтів більш гнучкою та зручною. Ось кілька загальних понять про CSS:

1. Основні поняття CSS:

Селектори визначають, до яких HTML-елементів застосовуються стилі. Основні типи селекторів:

  1. Тегові селектори: Вибирають елементи за тегом. Наприклад, p для всіх абзаців.
  2. Класові селектори: Вибирають елементи з певним класом, наприклад, .example.
  3. Ідентифікаторні селектори: Вибирають елементи з певним ідентифікатором, наприклад, #header.

2. Властивості та значення:

Властивості визначають, які стилі будуть застосовані, наприклад, color, font-size, margin. Значення вказують, яке значення має бути застосоване до властивості.

3. Блоки оголошень:

Кожен блок оголошення складається зі селектора та набору властивостей. Приклад:

CSS

p {
  color: blue;
  font-size: 16px;
}

4. Моделі коробок (Box Model):

CSS базується на моделі коробок, де кожен елемент веб-сторінки розглядається як прямокутна коробка, що складається з:

  1. Контент: Фактичний текст або зображення.
  2. Паддінг: Простір між контентом і межами елемента.
  3. Межа (Border): Лінія, що оточує елемент.
  4. Маргін (Margin): Простір між елементами.

5. Каскадність (Cascading):

CSS може містити кілька правил для одного й того ж елемента, і правила застосовуються у певному порядку. Визначаються пріоритети: інлайнові стилі мають вищий пріоритет, ніж стилі в тегу <style>, а стилі в CSS-файлі мають найнижчий пріоритет.

6. Медіа-запити:

CSS підтримує адаптивний дизайн через медіа-запити, які дозволяють змінювати стилі в залежності від розміру екрану, орієнтації тощо.

HTML

@media (max-width: 600px) {
  body {
    background-color: lightblue;
  }
}
